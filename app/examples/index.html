Building a Page from scratch

  • Step One - Base PHP

    The base PHP has the required elements of all pages on the site. This includes the metadata, page wrapper, header and footer.

    Base PHP Example
    <!-- Base PHP Example -->
<!doctype html>
<?php
    $Title = 'U.S. Energy Information Administration (EIA)';
    $pageTitle = ''; <!-- e.g. Contact Us or Petroleum Data -->
    $pageDescription = ''; <!-- Page description -->
    $pageKeywords = ''; <!-- Define keywords for search engines -->
    $pageSection =  '' <!-- e.g. about or petroleum-->

?>
<html>
    <head>
        <?php include('<!-- your path -->/global/head/includes/head.inc');?>
        <title><?=$pageTitle?> - <?=$Title?></title>
        <meta name="description" content="<?=$pageDescription?>">
        <meta name="keywords" content="<?=$pageKeywords?>">
    </head>
    <body>
        <?php include('<!-- your path -->/global/header/includes/header.inc');?>
        
        <!-- see the layouts section -->
        <?php include('<!-- your path -->/global/footer/includes/footer.inc') ?>
    </body>
</html>

  • Step Two - Page Layouts

    Layout rules divide the page into sections and apply the correct spacing (borders, margins and padding) and responsiveness (mobile, mobile-landscape, tablet, tablet-landscape, laptop, desktop and desktop-large). Layouts also define how page elements and modules render based on the users' screen resolution.

  • Step three - Modules

    Modules are the reusable, modular parts of our design. They are the callouts, the sidebar sections, the product lists and so on. Modules are going to be the bulk of any project. As a result, having every module start with a prefix like .module- would be needlessly verbose. Modules just use the name of the module itself.
