Categorizing CSS Rules

This style-guide is divided into five sections:

  • Base

    Base rules are the defaults. They are almost exclusively single element selectors but it could include attribute selectors, pseudo-class selectors, child selectors or sibling selectors. Essentially, a base style says that wherever this element is on the page, it should look like this.

    Examples of Base Styles

    html {
line-height: 1.15; 
	-ms-text-size-adjust: 100%; 
	-webkit-text-size-adjust: 100%; 
	margin: 0;
	padding: 0;
	width: 100%;
	font-size: 16px; 
	box-sizing: border-box;
}
* {
	box-sizing: border-box;
}
.print {
	display: none;
}
table {
	border-collapse: collapse;
	border-spacing: 0;
}

  • Layouts

    Layout rules divide the page into sections and apply the correct spacing (borders, margins and padding) and responsiveness (mobile, mobile-landscape, tablet, tablet-landscape, laptop, desktop and desktop-large). Layouts also define how page elements and modules render based on the users' screen resolution.

  • Modules

    Modules are the reusable, modular parts of our design. They are the callouts, the sidebar sections, the product lists and so on.

    Modules are going to be the bulk of any project. As a result, having every module start with a prefix like .module- would be needlessly verbose. Modules just use the name of the module itself.

    Examples of Layout Styles

    .basic-table {
	/* adds global styles for basic-table content */
}
.content-container {
	/* adds global styles for content containers */
}
.accordion {
	/* adds an accordion to the page (see states for options) */
	/* scroll-to anchors capable */
}
.fancybox-type-iframe { 
	/* adds a responsive modal for iframe content */
	/* scroll-to anchors capable */
}
.scroll-to-inline { 
	/* adds a responsive modal for inline content */
	/* scroll-to anchors capable */
}
.tabs {
	/* adds responsive tabs */
	/* scroll-to anchors capable */
}

  • States

    State rules are ways to describe how our modules or layouts will look when in a particular state. Is it hidden or expanded? Is it active or inactive? They are about describing how a module or layout looks on screens that are smaller or bigger. They are also about describing how a module might look in different views like the home page or the inside page.
