Naming Rules

By separating rules into the five categories, Base, Layout, Modules, States and Themes, the naming convention is beneficial for immediately understanding which category a particular style belongs to and its role within the overall scope of the page.

Base rules are the defaults. They are almost exclusively single element selectors but it could include attribute selectors, pseudo-class selectors, child selectors or sibling selectors. Essentially, a base style says that wherever this element is on the page.

Font size reset

The .font-size; will render as follows to the output .css file.

:root {
  font-size: 16px;
}

Default font

The .font; will render as follows to the output .css file.

{
  font-family: Arial, Helvetica, sans-serif;
  font-style: normal;
  font-weight: normal;
  color: #333;
}

Header font

The .header-font; will render as follows to the output .css file.

{
  font-family: "Times New Roman", "Times", serif;
  font-style: normal;
  font-weight: normal;
  color: #333;
}

The .bold; will render as follows to the output .CSS file.

{
  font-weight: 700; // bold
}

The .semi-bold; will render as follows to the output .CSS file.

{
  font-weight: 600; // semi-bold
}

The .normal; will render as follows to the output .CSS file.

{
  font-weight: normal;
}

The .right; will render as follows to the output .CSS file.

{
  float: right;
}

The .left; will render as follows to the output .CSS file.

{
  float: left;
}

How to use Global Variables and Mixins

For example you create a new class called .my-class and you want it to be a header font with a semi-bold weight and the color red. 

.my-class {
  .header-font;
  .semi-bold;
  color: red;
}

The .my-class calculated output the following styles:

.my-class {
  font-family: "Times New Roman", "Times", serif;
  font-style: normal;
  font-weight: 600;
  color: red;
  font-size: 1rem; /* browser default */
  line-height: 1.125em; /* browser default */
}
